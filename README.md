> **Ingeniería Electrónica & Embedded Systems.** Ecosistema de alto rendimiento optimizado para la exposición de proyectos de hardware e IoT, priorizando la velocidad de carga y la robustez mediante una arquitectura estática moderna y desacoplada.

---

## 🏗️ Arquitectura del Sistema (Static Performance)

Siguiendo principios de eficiencia industrial, el sistema ha sido refactorizado para eliminar latencias externas y dependencias de terceros:

- **Frontend**: Interfaz reactiva desarrollada en **Vanilla JavaScript** alojada en **GitHub Pages**. Implementa un motor de renderizado dinámico que procesa la lógica de UI sin frameworks pesados.
- **Capa de Datos (Local Sourcing)**: Gestión de proyectos mediante un esquema **JSON estructurado** (`docs/data/projects.json`). Esta aproximación garantiza disponibilidad del 100%, seguridad de datos y tiempos de respuesta de milisegundos (Zero Cold Start).
- **Procesamiento**: Lógica de internacionalización (i18n) y gestión de estados asíncronos integrada directamente en el core del cliente, optimizando el consumo de recursos.

## 🛠️ Stack Tecnológico

- **Firmware & Hardware Core**: C (XC8), PIC16F1939, KiCad 8.0, PCB Design.
- **Data & Automation**: Python 3.14 (Data Management), JSON, ETL Scripts para automatización de portafolio.
- **Frontend Core**: HTML5 Semántico, CSS3 (Arquitectura Modular & Glassmorphism), Vanilla JavaScript.
- **Herramientas de Ingeniería**: LaTeX (TikZ/LuaLaTeX), Makefile, Git (Conventional Commits).

## 🚀 Características de Ingeniería

- **Local JSON Cache**: Estrategia de datos desacoplada que permite actualizaciones rápidas sin necesidad de gestionar infraestructuras cloud complejas, ideal para entornos de portafolio técnico.
- **DIN-Style Layout**: Estructura visual inspirada en hojas de datos técnicos (Data Sheets), optimizando la jerarquía visual de KPIs y especificaciones técnicas.
- **V-Model Optimized**: Flujo de desarrollo y documentación alineado con metodologías de verificación y validación industrial.
- **Performance-First**: Optimización del _Critical Rendering Path_, eliminando el "bloqueo" por peticiones a APIs externas y priorizando el rendimiento nativo del navegador.

## 📋 Evolución del Proyecto

1.  **Migración a Arquitectura Estática**: Transición estratégica desde servicios externos (Supabase/Render) hacia una solución local para mejorar la confiabilidad y el SEO.
2.  **Estandarización de Esquemas**: Definición de objetos JSON rigurosos para la gestión de _Tech Stacks_, diagramas de progreso y metadatos de hardware.
3.  **Refactorización de Lógica**: Implementación de un sistema de `fetch` local optimizado con manejo de errores robusto.
4.  **Validación Documental**: Integración de documentación técnica avanzada generada en LaTeX directamente vinculada a los repositorios de hardware.

## 🎯 Enfoque Técnico

Como **Ingeniero Electrónico**, mi especialidad radica en cerrar la brecha entre el **Hardware de bajo nivel** y las **Arquitecturas de software eficientes**. Mi filosofía dicta que la robustez de un firmware debe reflejarse en la infraestructura que lo presenta: simple, rápida y trazable.

- **Especialidades:** Firmware Development (C/PIC), IoT Architecture, Documentación Técnica Industrial (LaTeX) y Automatización de Procesos.
- **Filosofía:** "La simplicidad es la máxima sofisticación". Diseño orientado a la eficiencia energética, la mantenibilidad y la precisión técnica.
